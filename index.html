Trilhando Rios

Plataforma de mapeamento geográfico, histórico e turístico do Tocantins

Objetivo Geral

Criar uma plataforma multimodal que integre dados geográficos oficiais, pontos turísticos e culturais do Tocantins, conteúdos jornalísticos e narrativos com pitadas artísticas, com operações automatizadas para ingestão, processamento e publicação de dados.

Objetivos Específicos

  • Construir pipeline automatizado de ingestão de dados via APIs e scrapers.
  • Centralizar dados em PostgreSQL + PostGIS, imagens em MinIO/S3 e textos em Elastic/Milvus.
  • Criar modelos multimodais para análise de imagens, OCR e geração de textos.
  • Desenvolver frontend interativo para visualização de mapas e artigos.
  • Implementar orquestração e automação completa dos processos.

Tecnologias Principais

Camada Ferramenta / Biblioteca Função
Backend Python + Flask API REST/GraphQL para MVP
Banco de dados PostgreSQL + PostGIS Geodados e consultas espaciais
Index/Busca Milvus / Elastic Busca semântica e RAG
Frontend React + Leaflet/Mapbox Mapas interativos e dashboards
IA / LLM OpenAI / Claude / Gemini (multimodal) Geração de texto + interpretação de imagens
ETL / Scraping Python (requests, BS4, Playwright, Geopandas) Ingestão e normalização
Armazenamento multimídia MinIO / S3 Fotos e vídeos com metadados
Orquestração Airflow / Prefect Jobs agendados e monitoramento

Roadmap de Execução

0

Fase 0 – Preparação

  • Definir escopo detalhado do projeto.
  • Criar repositórios Git e estruturas iniciais de pastas.
  • Configurar ambiente de desenvolvimento e teste (Docker).
1

Fase 1 – Ingestão e Armazenamento

  • Implementar scripts de ingestão de dados geográficos (IBGE, OSM).
  • Criar scrapers controlados para sites de turismo e cultura.
  • Estruturar banco PostGIS e buckets de mídia.
2

Fase 2 – Indexação e Pesquisa

  • Calcular embeddings de textos e imagens.
  • Indexar em Milvus/Elastic.
  • Criar endpoints básicos de busca e filtros internos.
3

Fase 3 – Geração de Conteúdo

  • Desenvolver pipeline RAG para textos com rigor jornalístico + interpretação artística.
  • Integrar análise multimodal (imagens → descrições/alt-text).
  • Testar prompts e fluxos de geração automática de artigos.
4

Fase 4 – Frontend e Visualização

  • Criar dashboards internos para revisão de conteúdos.
  • Mapas interativos com camadas de POIs, fotos e textos.
  • Ferramenta de curadoria rápida (aprovar/rejeitar conteúdo gerado).
5

Fase 5 – Automação Completa

  • Configurar jobs de ingestão periódica (diário, semanal, mensal).
  • Automação da reindexação e atualização de artigos.
  • Monitoramento de performance, erros e quotas de APIs.
6

Fase 6 – Testes e Ajustes Finais

  • Revisão interna de consistência factual e geográfica.
  • Ajustes de prompts e estilos de texto.
  • Testes de integração do backend + frontend + IA.

KPIs Internos / Métricas

  • % de cobertura geográfica do Tocantins (municípios e POIs).
  • % de fatos referenciados corretamente em artigos.
  • Tempo médio de geração de conteúdo por local.
  • Número de imagens aprovadas/licenciadas corretamente.
  • Disponibilidade da plataforma interna (>99% uptime).
